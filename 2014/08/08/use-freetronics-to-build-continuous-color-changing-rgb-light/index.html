Use Freetronics Experimenters Kit to Build Continuously Changing Color RGB Light

The Idea

When I went through all the parts inside Freetronics Experimenters Kit, and when I saw the RGB LED lying inside the kit, I had this idea that this LED continuously changes its color. I had this image in my mind that the color of the RGB LED just changes and never stop.

Demo Video

Because I used iPhone’s camera to capture the video, and the RGB LED is too bright for the camera(even after I set the maximum value of the color to 60 which is pretty low), the beautiful transition from one color to another can be barely seen in the video.

Video Link: http://youtu.be/MyuLrNblgDo

However, you can download the code from the Github repo which is mentioned below and try it on your own Experimenters Kit after you set up the hardware.

V3.2 Create The loop

V3.2.1

If we want the color changes from one to another gradually, then we need to:

  1. Have 2 sets of color values, one is current, another is the set of target colors. Already done!
  2. Every time only change the color values by a small amount.
  3. Test if current colors match target color once a while.
  4. If they do match, make a new set of target colors.
  5. Last but not least, delay for a while. Otherwise the LED will change its color like crazy. :P

Why using functions rather than value++; or values[index] += 1; to change the current value?

  1. It has a name which provides a explanation of the code inside.
  2. It’s easier to make changes (i.e. adding functionality or changing the step value, the “+1”) while keeping code clean.
  3. When changing the value it needs validation because the value is given to the chip through analogWrite(pin, value), where the value has a maximum value of 255, and a minimum value of 0.

Possible Pitfalls

Output to one pin at a time

If all the pins on RGB LED don’t get updated at the same time, and values/voltages on some pins don’t get updated, then the actual values/voltages will decrease. And that means if you don’t update the value on one pin, the color will change by itself because of decrease of the voltage on the pin.

Reading Values from OUTPUT pin

At the beginning of Version 3, I once thought, “Hey, why can’t I just get current values of RGB color by reading from the pins and compare those with my target values?”

FAILED.

Here’s why:

The pin modes of LED’s pins are all set to OUTPUT, which means it really just can’t read values IN. Well, I tried and it just doesn’t work this way.
